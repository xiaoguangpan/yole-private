**Date**: 2026-05-13
**Status**: Proposal — 设计完成，今 session 未实施
**Related**:
- 实施完成的手动 rename 见 [marathon session devlog](./2026-05-13-yole-rename-and-features-marathon.md) §13

## Context

实施手动 session rename（右键 → ✏️ 重命名 + inline 编辑 input）后讨论自动化扩展。JC 直觉是"LLM 根据内容自动 rename 更理想"——拆开后实际比手动复杂得多。

讨论的三种路径：

- **A 手动重命名**（已实施）：永远兜底，user agency 完全
- **B LLM 自动 rename**（被否）：turn N 后系统调 LLM。Cost surprise + 时机难拿 + 质量不可控 + 跟 A 状态机互斥逻辑复杂
- **C LLM 一键 rename**（待实施）：user 主动点 → LLM 出标题。复用 ⋯ 菜单基础设施，无 surprise，user agency 保持

JC 决定 **C 推到下次 session 实施**，今天先固化设计。

## Decisions

### 调用模型：默认 active LLM

会调用当前 session 在用的 LLM（LLMPill 显示的那个）。

| 候选 | 利弊 |
|---|---|
| ✅ active LLM（推荐） | 用户已配 + 跟 conversation 同质量 + cost 在同一 provider + 0 extra config |
| ⚠️ 专用轻量模型（如 Haiku） | cost 低但需 mykey.py 额外配置 + 多一个 provider + 失败点 |
| 🔮 Settings 可选（升级路径） | V0.3 dogfood 反馈驱动，工程量 ~30 LOC |

成本估算：3 轮对话 ≈ ~1500 tokens 输入 + ~20 tokens 输出 ≈ Sonnet 4.6 价位下 **< ¥0.05 一次**。可忽略。

### 实现路径：复用 `/btw` 同款机制

```python
# bridge 端伪码
def handle_rename_via_llm(self):
    backend = self.agent.llmclient.backend
    history_snapshot = deepcopy(backend.history[:6])  # 前 ~3 轮 Q+A
    prompt = build_rename_prompt(history_snapshot)
    wire = build_wire(backend, history_snapshot, prompt)
    title = ""
    for chunk in backend.raw_ask(wire):
        title += chunk
    title = clean_title(title)
    self._emit(SessionRenamedEvent(sessionId=self.session_id, title=title))
```

**关键约束**：
- **不写** `backend.history` —— 完全副本调用，对话上下文零污染
- **不阻塞** main run —— worker thread 跑，agent 即使在跑也能起 AI rename
- **single shot** —— 一次 LLM 请求出标题

### Prompt 设计

```
你是一个工作台助手。下面是用户和 AI 的对话片段。
请用 8-12 个中文字给这个会话起一个标题。

要求：
- 站在用户视角，作为"这个 session 是关于什么"的提醒
- 用名词短语，不要写成 "我们讨论 X" / "用户问 X" 这种 meta 描述
- 突出具体话题或行动目标，避免 "讨论 / 探讨 / 聊" 这类抽象动词
- 如果对话还很短不明朗，挑最具体的关键词

对话片段：
{前 3 轮 Q+A 拼接}

只输出标题本身，无任何前后缀、引号、解释。
```

**Prompt 设计要点**：
- 显式禁止 meta 措辞（用户问 / 我们聊 / 讨论）
- 要求名词短语而非句子
- 8-12 字限制避免太短无信息 / 太长 sidebar truncate
- 限定中文输出

**举例对比**（以"维特根斯坦 LLM 类比"那个对话为基准）：

| 风格 | 输出 | 评价 |
|---|---|---|
| 希望的 | `维特根斯坦与 LLM 类比` | ✓ 名词短语 + 具体话题 |
| LLM 容易给的 | `关于哲学的讨论` | ✗ 太泛 |
| LLM 容易给的 | `用户与 AI 探讨语言哲学` | ✗ meta |
| LLM 容易给的 | `维特根斯坦` | ✗ 太宽 |

Prompt 第一轮草案不一定能达到，可能要 dogfood 微调 2-3 次。

### UI 入口

加进已有的 TopBar `⋯` overflow menu（marathon devlog §12 实施）：

```
⋯ menu:
  🔄 重新注入工具
  ✨ AI 重命名 session  ← 新增（V0.2）
  🐱 Desktop Pet
```

**菜单项视觉状态**：

| 状态 | 视觉 |
|---|---|
| 默认 | ✨ AI 重命名 |
| 生成中 | ✨ ...重命名中（disabled，icon spinning）|
| 完成 | toast "已重命名为: {title}" + menu 项回到默认 |
| 失败 | toast 显示错误 + menu 项回到默认 |
| 会话还没开始（无 user message）| disabled，hover tooltip "对话还没开始" |

### 边界条件

- **会话没 user message** → menu item disabled
- **只有 1 条 user / 0 条 assistant** → 允许尝试但质量差（用户可重试）
- **生成中重复点击** → 第二次点击 no-op（防 race）
- **失败** → 原 title 不变，toast 错误
- **跟手动 rename 互动** → AI rename 同样是"非默认值"，之后 user message 不会自动派生覆盖；用户随时可手动改回

## Rejected Alternatives

- **Option B（auto-trigger 自动 rename）**：
  - cost surprise（每个 session 自动多一次 LLM call）
  - 时机难拿（turn N 哪个 N 合适？固定 vs 周期都有问题）
  - 质量不可控（LLM 容易给 meta / 太泛 / 太宽的标题）
  - 跟手动 rename 互斥逻辑膨胀（manually-named flag + auto-stopped flag）
- **专用 rename LLM**（不走 active）：V0.2 不做，V0.3 dogfood 驱动
- **Rename 触发只放 Cmd+K palette**：discoverability 差，新用户找不到
